Marquee GTM Operator

How to run the go-to-market agent — the complete walkthrough

No coding required. If you can open a terminal and type a sentence, you can run this. The one-page version is the cheat sheet; this is the full reference.

1 What this is

You have an AI operator that runs Marquee's go-to-market like a one-person marketing team. It knows Marquee's whole launch strategy, follows it every time, and remembers what it learns. You talk to it in plain English. It drafts posts, surveys, outreach, research, and plans — but it never sends, posts, or publishes anything without showing you first.

  • The skill = the playbook. The strategy, rules, voice, and step-by-step launch sequence.
  • The agent (operator) = the worker that reads the playbook and does the work, in Sarah's voice.

2 What's included

Everything lives in one folder inside the app: ~/Desktop/ROO/marquee/.claude/  (it's hidden in Finder because it starts with a dot — that's normal).

.claude/
├── agents/
│   └── marquee-gtm-operator.md  ← THE AGENT you talk to
└── skills/marquee-gtm/
    ├── SKILL.md  ← THE PLAYBOOK (strategy + rules + voice)
    ├── references/  ← deeper strategy detail
    └── knowledge/  ← THE MEMORY (edit these freely)
        ├── competitors.md
        ├── market.md
        ├── feedback.md
        ├── playbook-library.md
        └── decisions.md

You'll mostly only ever touch the knowledge/ files. Everything else you can leave alone.

3 How to run it (every time)

  1. Open the Terminal app.
  2. Type this and press Enter: cd ~/Desktop/ROO/marquee
  3. Type this and press Enter: claude
  4. You're now talking to Claude. Start your message with "Use the marquee-gtm-operator" and say what you want.

First time only: running claude may open a browser asking you to log in with the Anthropic account. Do it once and you're set. To leave: type /exit or close the terminal.

4 The prompts you'll actually use

Where do we stand?

Use the marquee-gtm-operator. Read the skill, figure out which stage the launch is on, and tell me the single most important next move. Ask me anything you need to know.

Test the Blueprint

Use the marquee-gtm-operator. Draft short outreach to 10–15 real people who are job-hunting or pivoting, asking them to try the Alignment Blueprint. Keep it in Sarah's voice. Don't send it — show me first.

First survey (own the data)

Use the marquee-gtm-operator. Draft our first survey so we start owning data. Pick the topic with the best PR and content potential, write the questions, and save the plan to the knowledge base.

Log feedback

Use the marquee-gtm-operator. Here's what a beta user said: "[paste]". Save it to the feedback file word-for-word, note the theme, and tell me if it changes anything.

Scout competitors

Use the marquee-gtm-operator. Research who else is competing with Marquee, rank them, note their weak spots, and save it to the competitors file.

You don't have to memorize these. Just describe what you want in normal English — the operator figures out the rest.

5 The one rule that keeps you safe

The operator will draft, research, build lists, and update its own memory files freely. But it will never post, email, DM, send, or publish anything in Marquee's name without showing you first and getting a yes. Anything going out to the public is always your call. So you can let it work without worrying it'll do something live behind your back.

6 How to add knowledge and feedback

Way A — just tell the agent (easiest)

In a claude session, say: "Use the marquee-gtm-operator. Add this to the competitors file: …" or "… A user said '[paste]' — log it in feedback." The agent writes it to the right file for you.

Way B — edit the file yourself

Open the file in any text editor and type at the bottom. The rule: add to the bottom, don't delete what's there, and put the date.

  • New thing a user said → feedback.md
  • A competitor launched something → competitors.md
  • A new stat to cite → market.md with a link to where it came from (no source = the agent won't use it)
  • A GTM idea you saw → playbook-library.md

7 Two kinds of changes — keep them separate

✓ Growing the knowledge

New feedback, a competitor move, a stat. Add it to the right knowledge/ file. Do this anytime, freely.

✕ Changing the strategy

The launch sequence, a core tactic, pricing. Edit SKILL.md on purpose and note why in decisions.md. Slow down for these.

Rule of thumb: adding facts = anytime. Changing the plan = slow down and write down why.

8 Saving your work (optional)

The agent and skill live in the app's code repository, so changes can be saved to a dated history with git. After meaningful changes:

cd ~/Desktop/ROO/marquee git add .claude git commit -m "GTM: short note on what changed"

If git isn't your thing, that's fine — the files are saved on the computer either way. This just keeps a history.

9 If something's not working

  • It doesn't act like the operator: make sure you started with "Use the marquee-gtm-operator," and ran claude from inside ~/Desktop/ROO/marquee.
  • You see "marquee-gtm" listed twice: harmless. The version inside this repo is the one to edit.
  • General check-up: run claude doctor in a terminal.
  • It cited a number with no source, or repeated a killed idea: tell it. The guardrails are market.md (sources) and decisions.md (settled calls).
